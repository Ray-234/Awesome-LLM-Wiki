# Contributing

Thanks for helping improve Awesome LLM Wiki.

This list is intentionally focused. A good entry should be directly useful for understanding, building, evaluating, or applying Karpathy-style LLM Wikis: persistent, agent-maintained, source-grounded Markdown knowledge bases.

## What Belongs Here

- Original writings, talks, or discussions that define the LLM Wiki pattern.
- Reusable templates, agent skills, schemas, prompts, MCP servers, or starter kits.
- Source ingestion tools that turn real-world documents or web sources into clean Markdown or structured data.
- Implementations and domain-specific LLM Wikis with public code, documentation, or concrete examples.
- Research on agent memory, RAG, long-context reasoning, knowledge graphs, and compounding knowledge systems.
- Evaluation, observability, and safety tools relevant to source grounding, citation quality, prompt injection, or wiki maintenance.

## What Does Not Belong Here

- Generic RAG tools with no clear relationship to LLM Wiki workflows.
- Closed-source products with only vague landing pages.
- Marketing posts without technical detail or reproducible workflow information.
- Duplicates, forks without meaningful changes, or abandoned placeholders.
- Resources that make unverifiable claims or cannot be accessed publicly.

## Entry Format

Use this format for most entries:

```md
- [Name](https://example.com) - One sentence explaining why it matters for LLM Wiki. `tag` `tag`
```

Keep descriptions short, factual, and non-promotional. Prefer explaining the role of the resource over repeating its tagline.

## Pull Request Checklist

- The link is public and reachable.
- The resource has a clear connection to LLM Wiki.
- The entry is placed in the most specific section.
- The description is one sentence.
- The tags are short and lowercase.
- The README remains alphabetically or thematically organized within the target subsection.
